SQL 명령어
- 데이터 정의어(DDL): 테이블 생성, 변경, 삭제
- 데이터 조작어(DML): 데이터 삽입,조회, 수정, 삭제
- 데이터 제어어(DCL): 데이터 접근 권한 부여, 제거
- 트랜젝션 제어어(TCL): 데이터 조작어 명령어 실행, 취소, 임시저장

- DBA: 데이터베이스 관리자, 기업 내에서 데이터베이스를 관리자
	데이터베이스 관리, 테이블 생성, 데이터 접근 권한 관리자
	데이터 정의어: 테이블 생성, 변경, 삭제
	데이터 제어어: 데이터 접근 권한 부여, 제거
	
- 데이터 분석가 역할: 데이터 분석을 통해, 새로운 인사이트를 도출
	데이터베이스 접근, 데이터 분석, 데이터 접근 권한 요청
	데이터 조작어: 데이터 삽입, 조회, 수정, 삭제
	트랜젝션 제어어: DML명령어 실행, 취소, 임시저장
	
- 데이터 정의어(DDL): 테이블을 생성, 열 추가, 열 데이터 타입변경, 테이블명 변경, 삭제할 때 사용하는 명령어
	테이블은 각 열마다 반드시 1가지 데이터 타입으로 정의되어야 합니다.
	회원테이블: 회원정보(숫자형), 이름(문자형), 가입날짜(날짜형), 수신동의(숫자형(논리형))
 데이블, 제약조건: 테이블은 각 열마다 제약 조건을 정의할 수 있습니다.
	1. PK(PRIMARY KEY): 중복되어 나타날 수 없는 단일 값, NOT NULL(NULL이없는 상태)
	2. NOT NULL: NULL이  허용하지 않음
	
- 데이터 조작어(DML): 데이터를 삽입, 조회, 삭제할 때 사용하는 명령어
	1. 삽입: 제약 조건 위반, 문자 및 날싸형('')
	2. 조회: 모든 열(*), 특정 열 이름 변경(AS)
	3. 수정: 특정 조건(WHERE)
	4. 삭제: 특정 조건(WHERE)
	
- 데이터 제어어(DCL): 데이터베이스 관리자(DBA)가 특정 사용자(User)에게 데이터 접근 권한을 부여 및 제거할 때 사용하는 명령어
	DBA 역할: 데이터베이스 관리, 테이블 생성, 데이터 접근 권한 관리자
	User 역할: 데이터베이스 접근, 데이터 분석, 데이터 접근 권한 요청
	1. 사용자 확인: USER MYSQL
	2. 사용자 추가 및 삭제: CREATE, DROP
	3. 권한 부여 및 삭제: GRANT, REVOKE

- 트랜젝션 제어어(DCL): 데이터 조작어 실행, 취소, 임시저장할 때 사용하는 명령어
				분할할 수 없는 최소 단위이며 논리적인 작업 단위
				실행(COMMIT): 모든 작업을 최종 실행
					실행은 회원테이블에서 트랜젝션을 시작(BEGIN)하고 회원테이블에 데이터를 삽입(INSERT)하고 해당 작업을 최종 실행(COMMIT)
				취소(ROLLBACK): 모든 작업을 되돌림
					취소는 회원테이블에서 트랜젝션이 시작(BEGIN)하고 회원테이블에 데이터를 삽인(INSERT)한 후 해당 작업을 되돌림(ROLLBACK)
				임시저장(SAVEPOINT): ROLLBACK 저장점을 지정하는 명령어
	1. 실행: COMMIT
	2. 취소: ROLLBACK
	3. 임시저장: SAVEPOINT
	
SQL 명령어 단원 정리
	- SQL 기본 명령어는 4가지
	데이터 정의어: 테이블 생성, 변경, 삭제
	데이터 조작어: 데이터 삽입, 조회, 수정, 삭제
	데이터 제어어: 데이터 접근 권한 부여, 제거
	데이터 트랜젝션:	데이터 조작어 명렁어 실행, 취소, 임시저장
	
	- 테이블, 데이터 타입 및 제약 조건
	회원테이블은 회원번호(숫자형-PK), 이름(문자형), 가입일자(날짜형-NOT NULL), 수신동의(숫자형)
